# Contributing to aesc

Thank you for being interested in contributing to aesc!

We welcome all kinds of contributions, including bug fixes, features, documentation improvements, typo fixes, and more. To keep the codebase healthy and reviews productive, here are a few guidelines:

1. For larger changes (roughly 100+ lines), please open an [issue](https://github.com/akaeli-aesc/aesc-cli/issues) to discuss the approach before you start. This helps us align on direction early and avoids wasted effort. Small fixes and improvements can go straight to a pull request.
2. We aim for high code quality. Please include tests where appropriate and run the linter (`ruff`) and type checker (`pyright`) before submitting. Maintainers may suggest changes during review.
