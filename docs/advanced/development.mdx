## Development Setup

### Option 1: Local Development

<Steps>
  <Step title="Install uv">
    ```bash
    curl -LsSf https://astral.sh/uv/install.sh | sh
    ```
  </Step>

  <Step title="Clone Repository">
    ```bash
    git clone https://github.com/akaeli-aesc/aesc-cli.git
    cd aesc-cli
    ```
  </Step>

  <Step title="Install Dependencies">
    ```bash
    uv sync
    ```
  </Step>

  <Step title="Run aesc">
    ```bash
    uv run aesc
    ```
  </Step>
</Steps>

**Limitations:**
- ⚠️ Security tools (nmap, sqlmap) not available
- ⚠️ Different environment than Docker
- ✅ Good for: Agent logic, tool development, testing

### Option 2: Docker Development

<Steps>
  <Step title="Clone Repository">
    ```bash
    git clone https://github.com/akaeli-aesc/aesc-cli.git
    cd aesc-cli
    ```
  </Step>

  <Step title="Build Image">
    ```bash
    docker build -t aesc:dev .
    ```
  </Step>

  <Step title="Run with Code Mount">
    ```bash
    docker run -it --rm \
      -v $(pwd):/opt/aesc \
      -v ~/.aesc:/root/.aesc \
      aesc:dev
    ```
  </Step>
</Steps>

**Benefits:**
- ✅ Full Kali environment
- ✅ All security tools available
- ✅ Same as production

## Creating Custom Tools

### Tool Interface

```python
from typing import ClassVar
from pydantic import BaseModel
from aesc.provider import CallableTool2, ToolReturnType
from aesc.tools.utils import ToolRejectedError

class ToolParams(BaseModel):
    """Parameters for your tool"""
    target: str
    option: str = "default"

class MyTool(CallableTool2[ToolParams]):
    """Your custom tool"""

    # Tool metadata
    name: ClassVar[str] = "MyTool"
    description: ClassVar[str] = "Description of what the tool does"
    params: ClassVar[type[ToolParams]] = ToolParams

    def __init__(self, approval, bash):
        """Dependency injection"""
        self._approval = approval
        self._bash = bash

    async def __call__(self, params: ToolParams) -> ToolReturnType:
        """Execute tool"""
        # Request approval
        if not await self._approval.request(
            self.name,
            "perform action",
            f"Action on {params.target}"
        ):
            return ToolRejectedError()

        # Your implementation
        result = await self._bash({
            "command": f"your-command {params.target}"
        })

        return result
```

### Example: nmap Tool

```python
from typing import ClassVar
from pydantic import BaseModel
from aesc.provider import CallableTool2

class NmapParams(BaseModel):
    target: str
    ports: str = "1-1000"
    scan_type: str = "sV"

class NmapTool(CallableTool2[NmapParams]):
    name: ClassVar[str] = "NmapTool"
    description: ClassVar[str] = "Network scanning with nmap"
    params: ClassVar[type[NmapParams]] = NmapParams

    def __init__(self, approval, bash):
        self._approval = approval
        self._bash = bash

    async def __call__(self, params: NmapParams):
        # Build nmap command
        command = f"nmap -{params.scan_type} -p {params.ports} {params.target}"

        # Request approval
        if not await self._approval.request(
            self.name,
            "run network scan",
            f"Scan {params.target}"
        ):
            return ToolRejectedError()

        # Execute via bash tool
        result = await self._bash({"command": command})

        # Parse results (optional)
        if result.get("success"):
            output = result["output"]
            # Parse nmap output...
            return {
                "success": True,
                "output": output,
                "parsed": parse_nmap_output(output)
            }

        return result
```

### Register Tool

Tools are registered by their fully-qualified `"module.path:ClassName"` string in
the agent's `agent.yaml`. There is no central registry or `__all__` to update — just
add the import path to the agent's `tools` list:

```yaml
# src/aesc/agents/default/agent.yaml
agent:
  tools:
    - "aesc.tools.file:ReadFile"
    - "aesc.tools.file:WriteFile"
    - "aesc.tools.bash:Bash"
    - "aesc.tools.my_tool:MyTool"  # Your new tool
```

## Creating Custom Agents

### Agent Specification

```yaml
# ~/my-agents/my-agent/agent.yaml
version: 1
agent:
  name: "my-agent"
  system_prompt_path: ./system.md
  tools:
    - "aesc.tools.file:ReadFile"
    - "aesc.tools.file:WriteFile"
    - "aesc.tools.bash:Bash"
    - "aesc.tools.my_tool:MyTool"
```

The system prompt lives in a separate markdown file referenced by
`system_prompt_path` (relative to the agent.yaml location):

```markdown
# system.md
You are a specialized security agent.

Your role:
- Specific task 1
- Specific task 2

Always:
- Get authorization
- Document findings
- Follow best practices
```

See the [Agents Reference](/api-reference/agents) for the complete schema.

### Agent Loading

Agents are loaded from a YAML file via `aesc.soul.agent.load_agent`. The CLI
resolves the agent file in this order:

1. The `--agent-file` CLI argument
2. The `AESC_AGENT_FILE` environment variable
3. The built-in default agent

```bash
aesc --agent-file ~/my-agents/my-agent/agent.yaml
```

## Testing

### Running Tests

```bash
# Run all tests
uv run python -m pytest tests/ -v

# With coverage
uv run python -m pytest tests/ --cov=src/aesc

# Specific test file
uv run python -m pytest tests/test_tools.py

# Specific test
uv run python -m pytest tests/test_tools.py::test_nmap_tool
```

### Writing Tests

```python
# tests/test_my_tool.py

import pytest
from aesc.tools.my_tool import MyTool, ToolParams

@pytest.mark.asyncio
async def test_my_tool_success():
    """Test successful tool execution"""
    # Setup
    approval = MockApproval(always_approve=True)
    bash = MockBash(return_value={"success": True})

    tool = MyTool(approval, bash)

    # Execute
    result = await tool(ToolParams(
        target="example.com",
        option="test"
    ))

    # Assert
    assert result["success"] is True
    assert "output" in result

@pytest.mark.asyncio
async def test_my_tool_rejection():
    """Test user rejection"""
    approval = MockApproval(always_approve=False)
    tool = MyTool(approval, None)

    result = await tool(ToolParams(target="example.com"))

    assert isinstance(result, ToolRejectedError)
```

```bash
uv run ruff check src/

# Fix auto-fixable issues
uv run ruff check --fix src/
```

### Type Checking

```bash
# Run pyright
uv run pyright src/

# Check specific file
uv run pyright src/aesc/tools/my_tool.py
```

### Style Guide

```python
# Good: Type hints
async def process_result(data: dict[str, Any]) -> str:
    """Process tool result"""
    return data.get("output", "")

# Good: Docstrings
class MyTool(CallableTool2[ToolParams]):
    """
    Custom tool for specific functionality.

    This tool provides...

    Example:
        tool = MyTool(approval, bash)
        result = await tool(ToolParams(target="example.com"))
    """
    pass

# Good: Error handling
try:
    result = await execute_command()
except CommandError as e:
    return {"success": False, "error": str(e)}
```

## Debugging

### Enable Debug Logging

```bash
export AESC_LOG_LEVEL=DEBUG
uv run aesc
```

### Add Debug Prints

```python
import logging

logger = logging.getLogger(__name__)

async def my_function():
    logger.debug("Function called")
    logger.info("Processing...")
    logger.error("Error occurred")
```

### Use pdb

```python
import pdb

async def my_function():
    pdb.set_trace()  # Breakpoint
    # Code continues...
```

### Docker Debugging

```bash
# Build with debug flag
docker build -t aesc:debug --build-arg DEBUG=1 .

# Run with logs
docker run -it --rm \
  -e AESC_LOG_LEVEL=DEBUG \
  aesc:debug
```
