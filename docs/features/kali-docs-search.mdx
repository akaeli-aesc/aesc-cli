## Built-in Documentation

aesc includes **290 Kali Linux tool documentation files**, making it easy to learn and use any security tool without leaving the CLI.

## How It Works

### Ask aesc About Tools

```bash
# Get documentation for any tool
aesc -c "how do I use nmap?"
aesc -c "show me sqlmap documentation"
aesc -c "what are the options for gobuster?"
```

### Search by Task

```bash
# Find tools by what you want to do
aesc -c "how do I scan for SQL injection?"
aesc -c "what tool finds subdomains?"
aesc -c "how to crack WPA2 handshakes?"
```

## Documentation Features

### Comprehensive Coverage

- **Tool Description** - What the tool does
- **Usage Examples** - Common commands
- **Options Reference** - All flags and parameters
- **Best Practices** - Security considerations
- **Common Pitfalls** - What to avoid

### Smart Search

aesc uses AI to find relevant documentation:

```bash
# Natural language search
aesc -c "I need to test a website for XSS vulnerabilities"
# → Finds: xsser, xsstrike, dalfox, OWASP ZAP docs

aesc -c "find hidden directories on a web server"
# → Finds: gobuster, dirb, dirbuster, ffuf docs
```

### Contextual Help

aesc provides context-aware documentation:

```bash
# During a scan
aesc -c "scan example.com"
# If something fails, aesc explains:
# "nmap failed: Permission denied. Run with sudo or use Docker."

# With tool suggestions
aesc -c "test SQL injection"
# aesc shows: "I'll use sqlmap. Here's what it does..."
# Then provides relevant sqlmap documentation
```

## Example: Learning nmap

```bash
aesc -c "teach me nmap"
```

**aesc responds with:**

```
nmap - Network Mapper

Basic Usage:
  nmap [Scan Type] [Options] {target}

Common Scan Types:
  -sS    TCP SYN scan (default)
  -sT    TCP connect scan
  -sU    UDP scan
  -sV    Version detection
  -O     OS detection

Examples:
  # Basic scan
  nmap 192.168.1.1

  # Scan specific ports
  nmap -p 80,443 example.com

  # Service version detection
  nmap -sV example.com

  # Full scan with OS detection
  nmap -A example.com

Want me to run a scan?
```

## Search Architecture

aesc uses a multi-layered search system:

1. **Keyword Matching** - Direct tool name or command search
2. **Semantic Search** - AI understands intent and context
3. **Category Filtering** - Searches within tool categories
4. **Usage Pattern** - Finds tools based on common tasks

## Documentation Format

Each tool doc includes:

```markdown
# Tool Name

## Description
One-line description of what the tool does

## Installation
Usually pre-installed in Kali

## Basic Usage
tool [options] target

## Common Options
-o    Output file
-v    Verbose mode
-h    Help menu

## Examples
Example 1: Basic usage
Example 2: Advanced usage

## Tips & Tricks
- Best practices
- Common mistakes
- Pro tips

## Related Tools
- Similar tools
- Complementary tools

## References
- Official website
- GitHub repository
- Additional resources
```

## Accessing Full Docs

### Within aesc

```bash
# Get full documentation
aesc -c "show full nmap documentation"

# Get specific section
aesc -c "show nmap scanning options"
aesc -c "show sqlmap injection techniques"

# Compare tools
aesc -c "compare gobuster vs dirb"
aesc -c "which is better: john or hashcat?"
```

### External Access

Documentation files are located at:
```
/opt/kali-docs/official-docs/
```

You can also access them directly:
```bash
# View with less
less /opt/kali-docs/official-docs/nmap.md

# Search with grep
grep -r "SQL injection" /opt/kali-docs/official-docs/

# Use the search tool
python3 /opt/kali-docs/kali_doc_search.py "XSS"
```
